Write JSDoc comment referencing: D-21 (queryKey namespace), D-20 (staleTime 0), Pitfall 4 (empresa null guard), Pitfall 2 (queryKey must start with 'transacoes' for auto-invalidation).

Export named function useFaturamento(year: number) returning an object.

Inside the function:
1. const { empresa } = useEmpresaStore() — reads from Zustand store hydrated by EmpresaProvider
2. useQuery call with: queryKey: ['transacoes', year, 'faturamento'], queryFn: () => transacaoService.getByYear(year), staleTime: 0, retry: false, enabled: !!empresa
3. Destructure: const { data: transacoes = [], isLoading, error } = useQuery(...)
4. Derive calculations after the hook call (not inside queryFn):
   - limiteAnual = calcLimiteAnual(empresa, year)
   - totalFaturado = calcTotalFaturado(transacoes)
   - percentual = calcPercentual(totalFaturado, limiteAnual)
   - projecao = calcProjecao(totalFaturado, limiteAnual, year)
   - alertaAtivo = calcAlertaAtivo(totalFaturado, limiteAnual, percentual)
5. Return object: { isLoading: isLoading || empresa === null, error, empresa, limiteAnual, totalFaturado, percentual, projecao, alertaAtivo }

Critical: the isLoading return value is `isLoading || empresa === null` (not just isLoading). This ensures the gauge shows skeleton while EmpresaProvider is still booting, even if the TanStack Query has technically "resolved" with empty data (because enabled was false). This implements Pitfall 4 mitigation.

No default export. Named export only (consistent with useTransacoes.ts pattern).
