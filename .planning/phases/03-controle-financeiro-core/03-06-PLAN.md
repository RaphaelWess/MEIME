## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Supabase | All transaction data flows through RLS-enforced Supabase queries |
| User tap → openSheet | tap event triggers openSheet(transaction) — transaction is server-returned data, already filtered by RLS |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-01 | Elevation of Privilege | FinancasTab transaction list | high | mitigate | List data comes from useTransacoes which calls transacaoService.getByMonth — RLS ensures only the authenticated user's rows are returned. Verified in T-03-01 Plan 02. |
| T-03-04 | Tampering / XSS | InicioTab / FinancasTab | medium | mitigate | All values rendered as JSX text nodes via centsToBRL() or t.categoria, t.data string slicing. No dangerouslySetInnerHTML allowed — enforced in task acceptance criteria. |
| T-03-W6-01 | Tampering | FinancasTab month navigator | low | accept | prevMonth/nextMonth are pure utility functions — no network call, no injection surface. Month/year state is local client UI state only. |
</threat_model>
