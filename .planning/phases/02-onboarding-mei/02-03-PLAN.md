  <action>
    Create src/utils/cnpj.ts with three exported functions:

    stripCnpj(value: string): string
    - Replace all non-alphanumeric characters using /[^A-Z0-9]/gi regex, uppercase the result
    - Accepts letters A-Z for alphanumeric CNPJ support (RESEARCH.md: production rollout July 6, 2026)

    formatCnpj(raw: string): string
    - Takes raw 0-14 char string (output of stripCnpj), returns masked display string
    - Progressive masking: 0-2 chars: raw; 3-5: XX.XXX; 6-8: XX.XXX.XXX; 9-12: XX.XXX.XXX/XXXX; 13-14: XX.XXX.XXX/XXXX-XX
    - Slice input to max 14 chars

    isValidCnpj(raw: string): boolean
    - Strip then check length === 14
    - Reject all-same-character sequences (regex /^(.)\1+$/)
    - Use ASCII-minus-48 modulo-11 algorithm (charCodeAt(i) - 48) — handles both numeric ('0'-'9' = 0-9) and alphanumeric ('A'-'Z' = 17-42) CNPJs
    - weights1 = [5,4,3,2,9,8,7,6,5,4,3,2] for first 12 chars; dv1 = sum1 % 11 < 2 ? 0 : 11 - (sum1 % 11)
    - weights2 = [6,5,4,3,2,9,8,7,6,5,4,3,2] for all 13 chars (12 + dv1); dv2 = sum2 % 11 < 2 ? 0 : 11 - (sum2 % 11)
    - Return s[12] === String(dv1) && s[13] === String(dv2)

    Create src/hooks/useCnpjMask.ts:
    - useState for raw (stripped, max 14 chars) and masked (formatted display)
    - handleChange(e: ChangeEvent<HTMLInputElement>): strips e.target.value, slices to 14, updates both states
    - Returns { raw, masked, handleChange }

    Create src/hooks/useOnboardingCnpj.ts:
    - useDebouncedCnpj: useState + useEffect with 500ms setTimeout to produce debouncedCnpj from rawCnpj
    - isReady: debouncedCnpj.length === 14 && isValidCnpj(debouncedCnpj)
    - useQuery({ queryKey: ['cnpj', debouncedCnpj], queryFn: fetchCnpjWithFallback, enabled: isReady, retry: false, staleTime: 1000 * 60 * 60 })
    - fetchCnpjWithFallback(cnpj: string): async function
      1. fetch BrasilAPI: https://brasilapi.com.br/api/cnpj/v1/${cnpj}
      2. If response.ok, return response.json()
      3. Else fetch OpenCNPJ: https://api.opencnpj.org/cnpj/${cnpj}
      4. If response.ok, normalize fields to BrasilAPI shape (see RESEARCH.md Pattern 2 — note: OpenCNPJ field names are ASSUMED, verify against live call and adjust mapping if needed)
      5. If both fail: throw new Error(brasilRes.status === 404 ? 'CNPJ_NOT_FOUND' : 'API_ERROR')
    - Do NOT set retry on the TanStack Query (retry: false) — fallback logic is inside queryFn
    - Export CnpjData type with: cnpj, razao_social, nome_fantasia, cnae_fiscal, cnae_fiscal_descricao, situacao_cadastral (integer from BrasilAPI), descricao_situacao_cadastral, data_inicio_atividade
