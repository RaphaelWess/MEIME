## Color

| Role | Value | Usage | Source |
|------|-------|-------|--------|
| Dominant (60%) | #FFFFFF / neutral-50 (#FAFAFA) | App background, screen surfaces, modal backgrounds | Default — white-dominant mobile app |
| Secondary (30%) | #F4F4F5 (zinc-100) | BottomNav background, card surfaces, input backgrounds, dividers | Default — neutral warm gray |
| Accent (10%) | #16A34A (green-600) | See reserved list below | Claude's discretion — Brazilian fintech green |
| Destructive | #DC2626 (red-600) | Delete account button, delete confirmation dialog text, error banners | Default — WCAG-compliant red on white |

**Accent reserved for (explicit list):**
1. Active BottomNav tab icon + label
2. Primary CTA buttons (Entrar, Criar conta, Salvar)
3. FAB (+) background
4. Focus ring on interactive elements
5. Auth links ("Esqueci minha senha", "Criar conta")

**Accent NOT used for:** passive text, card borders, dividers, placeholder text, secondary buttons, or informational content.

**Rationale for green-600:** #16A34A passes WCAG AA contrast (4.54:1 on white) for interactive elements. Green is the dominant color in Brazilian fintech (Nubank is exception; most MEI-focused apps — Conta Azul, ContaSimples, QuickBooks BR — use green for money). Aligns with product value of "faturamento saudavel".

## Interaction Contract

### Auth Flow

- **Entry point:** Welcome screen (not direct login). Audience (MEI) benefits from context before form. Welcome screen shows: app name, tagline "Gestão gratuita para MEI", single CTA "Começar agora" (leads to login/register tabs), privacy footer link.
- **Auth method:** Email + password (Supabase Auth). Magic link deferred — adds email dependency complexity for MVP.
- **Login form fields:** E-mail, Senha, "Entrar" button, "Esqueci minha senha" link, "Criar conta" link.
- **Register form fields:** E-mail, Senha (min 8 chars), "Criar minha conta" button, "Já tenho conta" link.
- **Post-login redirect:** Início tab (default active tab).
- **BottomNav visibility:** Hidden before login. Shown only inside authenticated shell.

### BottomNav

- Active tab: Inicio on first login. Active state: accent green icon + label (green-600).
- Inactive state: zinc-400 icon + label.
- BottomNav background: zinc-100. Top border: zinc-200 (1px).
- Tab icons (Lucide React, 24px): Home (Início), Wallet (Finanças), CalendarDays (Agenda), QrCode (Cobrar), User (Conta).
- Labels: 14px, weight 400. Always visible (no icon-only BottomNav).

### FAB

- Position: Fixed, bottom-right. Offset: 16px from right edge, 80px from bottom (clears BottomNav).
- Size: 56px x 56px, border-radius: 50% (circle).
- Background: green-600. Icon: Plus (Lucide, 24px, white).
- Shadow: `shadow-lg` (shadcn default large shadow).
- Phase 1 behavior: FAB is rendered but `onClick` is a no-op with `console.log("FAB: TransactionForm deferred to Phase 3")`. No visual feedback needed for Phase 1.

### Delete Account

- Location: Conta tab, below user info, labeled section "Zona de Perigo".
- Button style: Outlined (border), destructive color (red-600), full width on mobile.
- Confirmation: shadcn `AlertDialog` component. Default focus on "Cancelar" button.
- After deletion: Supabase `auth.admin.deleteUser()` (or user self-delete via RPC) + redirect to welcome screen.

### Privacy Route

- `/privacidade` — accessible without auth (public route).
- Layout: Simple scroll page, no BottomNav, back button ("Voltar") top-left.
- Content: Minimal LGPD-compliant privacy policy placeholder. Static text, no data fetching.
- Linked from: Auth welcome screen footer AND Conta tab "Privacidade" link item.

## Accessibility Requirements

- All interactive elements: minimum 44x44px touch target (iOS HIG)
- Color contrast: all text on backgrounds meets WCAG AA (4.5:1 for body, 3:1 for large text)
- green-600 (#16A34A) on white (#FFFFFF): 4.54:1 — passes AA
- red-600 (#DC2626) on white: 4.64:1 — passes AA
- Form inputs: always paired with `<Label>` via `htmlFor` / `id`
- AlertDialog: focus trapped inside modal, default focus on safe action (Cancel)
- BottomNav: `aria-label` on nav, `aria-current="page"` on active tab
