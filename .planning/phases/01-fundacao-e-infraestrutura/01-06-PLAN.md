    Create `vercel.json` in project root:
    - JSON object with a "rewrites" array containing one entry: { "source": "/((?!api/).*)", "destination": "/index.html" }
    - This fixes SPA deep-link routing (e.g. /privacidade direct load doesn't 404)

    Create `public/icon-192.png`:
    - A 192x192px PNG image
    - Content: a solid green (#16A34A) square with a white "M" letter centered in Inter font or similar sans-serif
    - Method: Use Node.js Canvas API or generate a valid PNG programmatically
    - Alternative approach if canvas API unavailable: create the file using a base64-encoded minimal green square PNG. The file MUST be a valid PNG with dimensions 192x192 — a zero-byte file or placeholder text file will fail Chrome PWA installability check
    - To generate programmatically via Node script (run once, not committed to the codebase):
      Write a temporary script `scripts/generate-icons.mjs` that uses the `canvas` npm package or writes a valid PNG binary. Run it: `node scripts/generate-icons.mjs`. Verify the output: `file public/icon-192.png` should show "PNG image data, 192 x 192"

    Create `public/icon-512.png`:
    - Same design as icon-192.png but 512x512px
    - Must be a valid PNG — Chrome installability requires both 192 and 512

    Create `.github/workflows/supabase-keep-alive.yml`:
    - Use the exact YAML structure from RESEARCH.md section "GitHub Actions Anti-Pause":
      - name: "Supabase Keep Alive"
      - on.schedule: cron '0 8 * * 1,4' (every Monday and Thursday at 08:00 UTC)
      - on.workflow_dispatch: {} (manual trigger)
      - jobs.ping.runs-on: ubuntu-latest
      - steps: single step using curl to GET ${{ secrets.SUPABASE_URL }}/auth/v1/health with header "apikey: ${{ secrets.SUPABASE_ANON_KEY }}", capture HTTP status code, fail if not 200
    - Secret names: SUPABASE_URL and SUPABASE_ANON_KEY (NO VITE_ prefix — these are GitHub Actions secrets, not Vite env vars; using VITE_ prefix here is Pitfall 7)

    Verify .gitignore does NOT list icon-192.png or icon-512.png (they must be committed).
    Verify .gitignore lists .env.local (must not be committed).
    Verify .gitignore lists node_modules/.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Secrets → GitHub Actions | SUPABASE_URL and SUPABASE_ANON_KEY stored as encrypted GitHub Secrets; not exposed in logs |
| Vercel env vars → client bundle | VITE_SUPABASE_URL and VITE_SUPABASE_ANON_KEY are bundled into JavaScript; anon key is public by design |
| Vercel HTTPS → browser | All traffic encrypted; required for PWA service worker registration |

| T-1-07 | Information Disclosure | GitHub Actions — secret naming confusion | low | mitigate | GitHub Secrets are SUPABASE_URL / SUPABASE_ANON_KEY (no VITE_ prefix — Pitfall 7 mitigation). Verified by workflow YAML acceptance criteria and manual run in Step 5. |
| T-1-08 | Denial of Service | Supabase free tier auto-pause | low | mitigate | GitHub Actions cron runs every Monday and Thursday (every 3 days) — well under 7-day inactivity threshold. Manual workflow_dispatch run in Step 5 verifies the workflow functions before first scheduled run. |
