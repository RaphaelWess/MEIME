<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| VITE_SUPABASE_ANON_KEY → bundle | Public key, safe in browser — but service_role must never appear here |
| authService.deleteAccount() → supabase.rpc('delete_user') | Calls SECURITY DEFINER function; only works for auth.uid() owner |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-1-02 | Elevation of Privilege | src/lib/supabase.ts — env var handling | high | mitigate | supabase.ts uses only VITE_SUPABASE_ANON_KEY; throws Error if missing; service_role key is never referenced. Verified by acceptance criteria: no VITE_SUPABASE_SERVICE_ROLE_KEY in any file. |
| T-1-04 | Repudiation | authService.deleteAccount() | high | mitigate | Calls supabase.rpc('delete_user') which is a SECURITY DEFINER function deleting WHERE id = auth.uid() — only the caller's own account. UI confirmation gate added in Plan 05 (AlertDialog). |
| T-1-03 | Information Disclosure | AuthProvider loading state | medium | mitigate | AuthProvider sets loading=true initially and false after getSession resolves. ProtectedRoute in Plan 04 uses loading state to avoid redirect flash — prevents auth state race condition. |
</threat_model>
