<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User form input → authService | Email/password from form fields; validated client-side (HTML5 minLength) + server-side (Supabase) |
| ProtectedRoute → /welcome redirect | Auth gate; loading state must be resolved before redirect fires |
| /privacidade (public route) | No auth required; must not be inside ProtectedRoute subtree |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-1-03 | Information Disclosure | ProtectedRoute — unauthenticated route access | medium | mitigate | ProtectedRoute checks user null + loading false before redirecting. All /app/* routes are children of this guard. /privacidade is explicitly a top-level public route (not inside /app). Verified by ProtectedRoute unit tests. |
| T-1-06 | Spoofing | AuthPage — no CSRF protection | low | accept | Supabase Auth uses JWT-based stateless auth; no server-side session cookie to forge. Supabase handles PKCE and token refresh. Acceptable for ASVS L1. |
</threat_model>
