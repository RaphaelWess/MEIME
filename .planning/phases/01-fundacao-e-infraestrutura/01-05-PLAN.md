## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ContaTab → authService.deleteAccount() | User-initiated account deletion; must only delete the authenticated user's own data |
| AlertDialog confirmation gate | UI gate before destructive action; Cancel is safe default |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-1-04 | Repudiation | ContaTab — delete account confirmation | high | mitigate | AlertDialog with explicit "Sim, excluir minha conta" confirm button required; Cancel is Radix default focus; only authService.deleteAccount() called on confirm (which calls SECURITY DEFINER delete_user() — deletes only auth.uid() row). No auto-confirmation possible. |
| T-1-03 | Information Disclosure | AppShell — BottomNav visibility | medium | mitigate | AppShell is only rendered as child of ProtectedRoute (established in Plan 04 router). BottomNav is inside AppShell — it is structurally impossible to render BottomNav without auth. Verified by router tree in App.tsx. |
</threat_model>
